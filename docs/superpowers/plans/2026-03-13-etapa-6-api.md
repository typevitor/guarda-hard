# Etapa 6 - API Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implementar a Etapa 6 da PRD com controllers, services (use cases), CRUDs de departamentos/usuarios/hardwares e endpoints de emprestimo, devolucao, defeito e conserto com isolamento multi-tenant e validacao de payload.

**Architecture:** Seguir arquitetura modular do `docs/architecture.md`: cada modulo com camadas `application`, `domain`, `infrastructure`, `presentation`. Controllers HTTP ficam finos, casos de uso na camada `application`, regras de negocio nas entidades de dominio, e persistencia via repositorios ja existentes com `TenantContext`.

**Tech Stack:** NestJS 11, TypeORM 0.3.x, SQLite, Zod, Vitest, TypeScript strict, pnpm workspaces

---

## Scope Check

Etapa 6 e um subsistema unico (API), mas com 3 trilhas paralelas naturais:

1. controllers (camada HTTP)
2. services/use-cases (camada application)
3. testes (unitarios de use-case + integracao HTTP)

Este plano mantem as 3 trilhas no mesmo branch para entregar um incremento funcional unico da Etapa 6.
